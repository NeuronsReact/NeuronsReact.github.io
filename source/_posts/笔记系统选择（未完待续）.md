具有通用格式文件、能够“find in content”、全平台同步的笔记系统是整个资料管理系统的“索引”。笔者之前的方案是 Obsidian + LiveSync + C2 存储，但今天 LiveSync 又又又炸了。为了不影响资料管理，笔者第一时间给 Obsidian 充了钱，不过每月 35CNY 的价格，也不算便宜，而且后期单个 vault 肯定不够，因此笔者又踏上了寻找解决方案的旅程。
<!-- more -->

首先想到的是老相识 tiddlywiki。Windows 端 timimi + 浏览器 + OneDrive 和 Android 端 tiddloid + OneDrive 是笔者更早以前使用的方案，但单文件形式的 tiddlywiki 保存一次就要重传整个 html，也无法多端同时编辑，体验并不算好。林一二先生开发的 TidGi 桌面端和移动端使用了 NodeJS 形式 tiddlywiki，可以单个 tid 传输，并内置了 GitHub 同步。唯一的缺点是只支持 GitHub 同步，对网络要求较高。
如果只考虑 Markdown 基本编辑需求，桌面端可以用 VSCode 直接编辑网盘文件或 GitHub 库，而移动端只要能解决 SAF 访问网盘文件夹或基本 Git 操作和 Markdown 渲染就行。GitSync + Obsidian 似乎不错，但仔细一看 GitSync 的作者也不建议在移动端使用，已经预感到到移动端上奇怪问题一堆的情形了，而 OneDrive、GoogleDrive 都没做 SAF 文件夹 API 的适配（seafile app 似乎支持 SAF 文件夹 API？）。最优的方案似乎只有以上 3 种。
目前打算先试试 TidGi Mobile，毕竟是较多用户使用过的，还能把以前没迁移 Obsidian 的旧笔记合入。
